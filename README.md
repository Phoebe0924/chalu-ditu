# 非标机会生成器（Internal V0.1）

一个面向非标履历人群的本地机会工作台。它不以“生成一份漂亮求职报告”为终点，而是帮助用户：

```text
理解自己
→ 识别可证明价值与边界
→ 找到具体对象
→ 设计低风险入口
→ 发出真实行动
→ 记录外部反馈
→ 校准对象、开口、证据或合作方式
```

首个垂直只覆盖职业岗位、项目试单、创始人合作和弱关系引荐。系统不会自动联系任何人，也不做实时面试辅助。

## V0.1 完成的闭环

### 1. 结构化机会评估

系统分别判断六条机会路径：

- 标准岗位路径
- 提案式求职路径
- 作品集吸引路径
- 弱关系引荐路径
- 试单/项目路径
- 创始人约聊路径

每条路径返回排名、优先级、适配信号、风险、证据缺口、推荐理由和暂不优先条件。

### 2. 事实—价值—证据—边界账本

每项市场价值都必须回溯到用户提供的事实和证据，并说明：

- 能证明什么
- 不能证明什么
- 还缺什么证据
- 对外表达的边界

### 3. 可编辑行动资产

根据优先路径生成 3–5 份独立资产，例如：

- 创始人私信
- 弱关系消息
- 两周试点提案
- 作品集结构
- 低风险试单
- 20 分钟约聊提纲
- 岗位匹配与面试准备

每份资产可编辑、复制并人工确认。只有人工确认后的资产才能关联到待发送行动。

### 4. 目标与真实反馈管线

- 建立目标对象池
- 记录具体联系理由、问题假设和研究证据
- 跟踪研究、待发送、已发送、回复、约聊、试点和关闭状态
- 每次行动保存当时使用的材料快照，后续改稿不会覆盖历史
- 记录真实反馈类型与原文
- 每个未回复对象最多记录一次带新信息的跟进
- 根据反馈定位失败层级并生成一个最小修正

### 5. 两周冲刺指标

工作台顶部固定显示：

- 研究目标：10
- 真实发送：6
- 有效机会进展：2

有效进展包括实质性回复、引荐、愿意看作品、约聊、面试、讨论试点或提出合作条件。

## 产品护栏

- 不伪造经历、数据、客户、收入或技术能力
- 不夸大成果，不鼓励卖惨
- 不把照护、生育或空窗硬包装成商业成果
- 不把会使用 AI 工具的人自动描述为工程师
- 不自动发送消息或提交申请
- 所有价值主张必须引用证据账本

## 技术架构

- Next.js App Router
- TypeScript
- Tailwind CSS
- OpenAI Responses API
- GPT-5.5 原生 Structured Outputs
- 版本化浏览器 `localStorage`

核心接口：

| 接口 | 作用 |
| --- | --- |
| `POST /api/assess` | 返回结构化 `OpportunityAssessment` |
| `POST /api/action-package` | 根据已确认评估返回 `ActionAsset[]` |
| `POST /api/calibrate` | 根据一次真实行动与反馈返回 `FeedbackDiagnosis` |
| `GET /api/regression-samples` | 返回六类固定回归样本 |

Markdown 只作为导出格式，不再是内部数据源。

## 本地运行

要求 Node.js 20.9 或更高版本。

```bash
npm install
cp .env.example .env.local
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)。

生产检查：

```bash
npm run lint
npm run build
npm start
```

## API Key

`.env.local`：

```dotenv
LLM_API_KEY=你的密钥
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-5.5
```

- 也兼容读取 `OPENAI_API_KEY`
- `LLM_BASE_URL` 必须支持 `/responses`
- 三个核心接口默认使用中等推理强度，并对瞬时网络中断自动重试一次
- 三个接口都使用严格 JSON Schema 输出，并保留业务规则二次校验
- 密钥仅在服务端读取
- 未配置密钥时，仍可使用本地目标与行动记录

## Phoebe 两周实战

首轮固定场景：

> 联系 AI 应用创业公司创始人，围绕案例、教程和用户场景表达问题，尝试进入约聊或两周试点。

建议节奏：

| 时间 | 行动 |
| --- | --- |
| 第 1–3 天 | 完成事实档案、路径评估、行动资产和目标记录 |
| 第 4–5 天 | 建立并研究 10 个对象 |
| 第 6–8 天 | 完成前 3 次发送 |
| 第 9 天 | 根据真实反馈校准 |
| 第 10–12 天 | 完成后 3 次发送 |
| 第 13 天 | 唯一一次有新信息的跟进与约聊准备 |
| 第 14 天 | 产品与机会双重复盘 |

阶段通过条件：

- 10 个对象均有具体联系理由和问题观察
- 完成 6 次真实发送
- 至少 2 次有效机会进展
- 至少 1 次约聊、引荐、作品审阅或试点讨论
- 所有发送内容均能回溯到真实证据
- 从输入到可发送行动包不超过 30 分钟

## 六类回归样本

页面的“事实输入”区可以载入：

1. 妈妈重返职场
2. 长期照护空窗
3. 传统行业转行
4. 自由职业转全职
5. 创业失败后重新进入市场
6. 普通人向 AI 相关工作转型

每个样本都定义了应优先路径、应降级路径、必要风险、禁止声明、必要行动材料和补证要求。

使用真实模型运行 18 次回归测试：

```bash
# 终端 1
npm run dev

# 终端 2
npm run test:regression
```

可选变量：

```bash
REGRESSION_BASE_URL=http://localhost:3000
REGRESSION_RUNS=3
REGRESSION_SAMPLE_ID=mother-ai-return
```

通过标准：

- 总通过率至少 90%
- 高风险正向身份声明失败数必须为 0
- 每个样本默认运行 3 次

## 本地数据

工作区使用 key：

```text
nonstandard-opportunity-workspace-v2
```

保存内容包括档案、评估、行动资产、目标对象、行动记录和反馈诊断。当前没有登录、数据库或云同步，清除浏览器站点数据会删除工作区。

## 已知限制

- 当前只支持单用户、单工作区
- 没有多人陪跑审核界面
- 没有岗位抓取、简历 PDF 解析或自动投递
- 模型输出质量仍需用真实回归集持续校准
- 本地指标不能证明长期就业结果，只用于首轮机会闭环验证
- “人生机会 OS”是长期愿景，不是当前产品承诺
