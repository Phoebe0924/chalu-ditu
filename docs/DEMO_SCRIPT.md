# 岔路地图 · 3 分钟演示讲稿（MVP V0.2）

演示准备：打开应用，点右上角「演示闭环」→ 确认载入。页面会直接停在「闭环复盘」视图。

演示数据说明（必须先讲）：这份闭环是系统真实生成的一次完整记录——评估、行动资产、校准全部来自真实模型调用；其中目标对象「陈屿 / 折光智能」和他的回复是虚构示例，用于演示流程，不是真实客户案例。

---

## 0:00–0:30 一句话定位

> 「岔路地图服务的是非标履历的人：空窗、生育、转行、创业失败。它不帮你把简历写得更厉害，它帮你完成一次真实的机会推进——并且全程不允许伪造经历。」

指着顶部进度条：输入事实 → 判断路径 → 行动资产 → 目标对象 → 完成行动 → 记录反馈 → 校准下一步。七个勾，全部来自真实操作记录，系统没有任何一步可以凭空打勾。

## 0:30–1:10 前半段：从事实到可发送材料

切到「事实输入」标签，快速滚动：

> 「用户输入的是事实底稿：6 年运营经验、三年育儿空窗、一个 LightPic 项目。注意最后一栏是『不希望被如何包装』——用户自己定叙事底线。」

切到「路径评估」：

> 「系统没有推荐海投。六条路径里，提案式求职、创始人约聊、作品集吸引排前三，标准岗位投递排最后，并说明了为什么。」

滚到证据账本：

> 「关键在这里：7 条证据分了验证等级。『6 年经验』没有外部材料，被标为孤证——孤证会被代码层挡在所有对外文案之外。这不是 prompt 约束，是服务端规则。」

## 1:10–1:50 后半段：真实行动与校准

切到「行动资产」：

> 「4 份可编辑材料，每份都带主张核查：每句话能回溯到哪条证据、什么等级、表达边界在哪。私信开头是『我想请教一个很小的问题』，不是『我是资深专家』。」

切到「目标与反馈」：

> 「用户把材料复制出去、自己发送——系统从不代发。对方回复进入试点讨论后，用户录入原文，点校准。」

指着校准结果：

> 「校准没有重写报告，它定位在『推进层』，给了一个今天就能做的动作：通话前准备一页试点草案，写清交付件、验收标准和报价底线。」

## 1:50–2:40 商业闭环：为什么值得付费

切回「闭环复盘」：

> 「这页是产品的商业答案。价值账单全部来自真实记录：7 条证据、2 条孤证被拦、15 条主张核查、从输入到可发送材料 7 分钟。」

滚到付费区：

> 「免费额度刚好够跑通第一个闭环：评估 2 次、行动包 2 次、校准 4 次。用完后代码层直接拦截调用。想推进第二个、第三个目标——¥49 内测买断本机 Pro，解锁码由服务端 HMAC 校验，伪造码会被拒绝。」

> 「定价页只承诺一件已实现的事：解除额度限制。没有虚假的企业版功能列表。」

## 2:40–3:00 收尾

> 「大多数求职工具在『生成一份更好的材料』就结束了。岔路地图的闭环终点是：你知道下一步是什么，并且看得见这个工具在每一步替你守住了真实性。这就是它敢收费的原因。」

## 常见追问备答

- **额度存本机，清浏览器不就绕过了？** 是的，README 公开承认这一点。内测阶段先验证「有人愿意为闭环付费」，再上账号体系。
- **演示里的创始人是真的吗？** 不是，是虚构示例，载入时的确认弹窗已注明。系统生成的评估、资产、校准是真实模型输出。
- **和 AI 简历工具的区别？** 简历工具优化表达，岔路地图约束表达（孤证排除、主张核查）并推动真实行动闭环。
