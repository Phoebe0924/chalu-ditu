# Agent 测试说明

## 项目定位

这是「非标机会生成器」Internal V0.1。它帮助非标履历用户完成：

1. 事实与证据整理
2. 六条机会路径评估
3. 可直接使用的行动资产生成
4. 目标对象与行动状态记录
5. 根据真实回复进行反馈校准

请把它当作内部验证原型，不要按公开 SaaS 产品标准推断尚未承诺的功能。

## 产品边界

- 不伪造经历或成果
- 不把用户包装成并非真实具备的身份
- 不自动发送消息或申请
- 不做实时面试监听或面试外挂
- 每项价值主张应能回溯到事实或证据

## 推荐测试路径

1. 打开首页，选择“妈妈重返 · AI 产品运营”回归样本。
2. 检查事实输入和机会场景是否清楚。
3. 生成结构化机会评估。
4. 检查六条路径是否各出现一次，且提案式求职、创始人约聊、作品集吸引优先。
5. 检查证据账本是否出现三种验证等级，且每项都有具体判定依据和补证行动。
6. 生成行动资产。
7. 检查是否包含创始人私信、两周试点提案和约聊准备，并且没有引用“自述—孤证”。
8. 新建目标对象与行动记录，录入一次真实或模拟反馈。
9. 检查反馈校准是否只给出一个明确的下一步。

## 核心接口

| 接口 | 作用 |
| --- | --- |
| `POST /api/assess` | 生成 `OpportunityAssessment` |
| `POST /api/action-package` | 生成 `ActionAsset[]` |
| `POST /api/calibrate` | 生成 `FeedbackDiagnosis` |
| `GET /api/regression-samples` | 获取六类固定样本 |

## 重点审查问题

- 路径判断是否真正考虑非标准机会，而非默认投递 JD？
- 是否明确指出空窗、证据不足、非技术背景等风险？
- 是否出现虚构经历、商业结果或身份拔高？
- “6 年经验”等无材料支撑的单点自述是否被降为孤证？
- LightPic 的明确产品与过程材料是否被识别为可验证或内部一致？
- 孤证是否被服务端排除在行动包上下文之外？
- 行动材料能否直接修改后发送？
- 反馈校准是否基于实际反馈，而非重新生成一篇泛化报告？
- 用户能否在每一步知道下一个最小行动？

## 已知限制

- 单用户、单工作区
- 数据仅保存在浏览器 `localStorage`
- 没有登录、数据库或云同步
- 部署地址会消耗项目所有者的模型 API 额度，请避免重复批量生成
- 当前只验证职业岗位、项目试单、创始人合作和弱关系引荐
