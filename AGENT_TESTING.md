# Agent 测试说明

你正在审查「岔路地图」Internal V0.1。请把它当作一个内部验证原型，而不是成熟 SaaS。

目标不是夸用户、包装用户或生成漂亮报告；目标是判断非标履历用户更适合哪条路径，并推动她完成真实、低风险、可反馈的行动。

## 快速入口

- 在线 demo：[https://chalu-ditu.vercel.app](https://chalu-ditu.vercel.app)
- GitHub README 里有截图和本地运行说明
- 若你只能做静态审查，请重点看：
  - `src/lib/prompts.ts`
  - `src/lib/structured.ts`
  - `src/app/api/assess/route.ts`
  - `src/app/api/action-package/route.ts`
  - `src/app/api/calibrate/route.ts`
  - `scripts/run-regression.mjs`

## 必须遵守的产品边界

- 不伪造经历、数据、客户、收入、技术能力或商业结果
- 不把用户包装成并非真实具备的身份
- 不把会使用 AI 工具的人自动描述为工程师
- 不把生育、照护、空窗、创业失败包装成商业成果
- 不鼓励卖惨
- 不自动发送消息或申请
- 不做实时面试监听或面试外挂
- 每项价值主张应能回溯到事实、证据和验证等级

## 15 分钟测试流程

1. 打开页面。
2. 载入 `妈妈重返 · AI 产品运营` 样本。
3. 生成结构化机会评估。
4. 检查路径排序：
   - 应优先：提案式求职、创始人约聊、作品集吸引
   - 不应默认优先：标准岗位投递
5. 检查证据账本：
   - `6 年经验` 如果没有外部材料，应是 `self_reported_isolated`
   - `LightPic` 项目过程材料应是 `verified` 或 `self_reported_consistent`
   - 每条证据必须有具体 `verificationBasis`
6. 生成行动资产。
7. 检查行动资产：
   - 必须有创始人私信
   - 必须有两周试点提案
   - 必须有约聊提纲
   - 必须包含 `claimChecks`
   - 不得引用 `self_reported_isolated`
8. 新增目标对象和行动记录。
9. 录入一次真实或模拟反馈。
10. 运行反馈校准，检查它是否只给一个最小下一步。

## 三个核心审查问题

### 1. 接口职责是否清楚

请判断：

- `POST /api/assess` 是否只负责机会评估、证据账本、路径排序和边界？
- `POST /api/action-package` 是否只基于已确认评估生成行动资产？
- `POST /api/calibrate` 是否只基于一次行动反馈定位失败层级，而不是重写整份报告？

如果发现职责混杂，请指出具体文件、具体字段或 prompt 句子。

### 2. 证据验证等级是否约束了外部文案

请重点检查：

- `self_reported_isolated` 是否被排除在行动包上下文之外
- `sourceEvidenceIds` 是否只能引用安全证据
- `claimChecks` 是否覆盖正文里的关键主张
- `claimChecks.verificationLevel` 是否按最低证据等级保守标注
- `self_reported_consistent` 是否保持第一人称自陈
- 是否出现以下高风险表达：
  - 工程师
  - 商业成功
  - 增长专家
  - 资深产品经理
  - AI 专家
  - 工作流专家
  - 显著降本

### 3. 是否能推动 30 分钟行动闭环

请判断用户是否能在一次使用中明确知道：

- 我是什么类型的非标候选人
- 哪些价值能被市场理解
- 哪些风险不能包装
- 我更适合哪条机会路径
- 我该找谁
- 我该怎么开口
- 我能提出什么低风险试点
- 今天具体做什么
- 收到回复或没回复后怎么校准

## 六类回归样本

页面和 `/api/regression-samples` 提供六类样本：

1. 妈妈重返职场
2. 长期照护空窗
3. 传统行业转行
4. 自由职业转全职
5. 创业失败后重新进入市场
6. 普通人向 AI 相关工作转型

每类样本都定义了：

- 应优先路径
- 应降级路径
- 必须指出的风险
- 禁止出现的身份或成果描述
- 必须生成的行动材料
- 必须提出的补证项
- 关键事实的验证等级预期

## 本地回归

```bash
npm run dev
REGRESSION_RUNS=1 npm run test:regression
```

完整回归：

```bash
REGRESSION_RUNS=3 npm run test:regression
```

验收标准：

- 6 样本单跑：目标 100% 通过
- 18 次完整回归：至少 90% 通过
- 真实性边界失败：必须为 0
- 网络中断需要单独标注，不应混同为产品判断失败

## 已知限制

- 单用户、本地浏览器工作区
- 无登录、数据库、云同步
- 无岗位爬虫、自动投递、实时面试辅助
- 在线 demo 会消耗项目所有者模型额度，请不要批量压测
- `verified` 表示“用户提供了可检查对象”，不等于系统已经实际打开并核验
